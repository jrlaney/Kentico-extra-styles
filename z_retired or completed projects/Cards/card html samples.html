Overview

Cards are a layout and organizational tool for grid and list-based content on the website. They are generally used in two ways: to display related information that is ancillary to the main content of a page (such as homepage mini-banners, related resources, featured content, or contact lists), or on list-template pages to display a large filterable grid of sub content (such as the resource library, customers, events, or education courses pages.) More rarely, in some instances the filterable card grid can be used to organize a sub-section of content on a static content page (such as the data connector or consulting offers lists on their respective pages).

Cards have a standard look-and-feel on the website, created by unique HTML markup and a specialized set of classes. These standards make sure that the look of the cards is maintained across the many pages, contexts, and applications in which they are used. This guide details how to conceptualize, layout, and code the standard card object, and has example cards from across the website to serve as illustration or inspiration for how to layer content into a card appropriately.

Anatomy of a Card

A card is an article element usually made of 4 stacked sections: a header with a type and icon, 1-2 content sections for image and text, and a footer with a CTA link button. Whether each of the sections is used depends on the location, content, and purpose of the card.

Card header

The card header is an optional section that can go at the top of the card element. It can be used to display an icon or date (month/day format) in the upper left corner. In a text-only card, the type label can also be moved into the card header to save on vertical space. The card header should not be used over a resource-thumbnail image, and the type label should not be in the card header over any kind of image.

Contents and options:

  • Card corner: This element is mandatory when making a card header, and is used to create a square in the upper left of the header. It should denote the card type with an icon, or show a calendar date. Avoid placing plain text in the card corner.
  • Type Label: The type label text is optional in the header, but can be displayed to the right of the card corner. If the card header is positioned over image content, the type label should not be used in the header and should instead appear with the text content above the card title.

Card Content - Image

The card content section(s) within each card are used to display text and image content. Image content is optional, and is usually displayed at a 16:9 ratio block in the first content section. The image is generally either a logo, a plain photographic image with a light overlay/hover effect, or a standard resource thumbnail element with icon and CTA. If the card has a CTA link in the footer, the image should also be linked to the same destination. If the cards are in a list or grid, they must all use the same image treatment for image content in this section, for instance, resource thumbnails and simple images should not be mixed in the same group of cards, you must choose one or the other depending on what is most appropriate for the group.

Contents and options:

  • Simple image: A simple photographic image is generally used when the card is being linked to a non-resource page or website section (for instance, linking to a 'call for speakers' page or to 'explore the intelligent enterprise' microsite), or when resources and other content cards are mixed in the same list.
  • Resource Thumbnail: A resource thumbnail (image + icon + CTA) element is used when the card is linked to a specific resource library page, document, or video.
  • Logo: When referencing a company, the company's logo can be used in the image section either in conjunction with a background image (in the case of linking to a customer case study page) or by itself on a white background.

Card Content - Text

The card content section(s) within each card are used to display text and image content. The text content is located below the image (if it exists) and has several display and sub-content options depending on the purpose of the card. Most cards will include the type label, title, and description, and the content can be expanded with extra details, tags, and additional CTA links.

Contents and options:

  • Card type: The type label element should appear above the card title when it is present, except in text-only cards where is can me moved into the card header.
  • Card title: Generally, the minimum text content for a card is the card title, which is used to denote what resource title, page name, or section the card is associated with.
  • Line: The line element should appear below the card title to separate it from the rest of the text content. The line should be removed if there is no additional content after the title.
  • Extra Details: The extra details for a card might include: additional filter dropdown options beyond the main card type, duration, related company, byline, author, or other ancillary information about the card that is not a keyword tag. There is no hard limit for how many extra details can be included, but between 1-3 is ideal.
  • Card description: A paragraph or two of text describing the content.
  • Additional CTA links: Any extra CTA links beyond the main link in the footer should appear as text links below the description. Note that the links should be within a separate paragraph element and have a &raquo; HTML character at the end, as is the standard for stand-alone text links on the website.
  • Tags: The bottom of the text content section can include any number of tags. These usually correlate to keyword tags that have been entered for the content from a database, but can also be used to display extra information not appropriate for the extra details lines.

Card Footer

The card footer is the final section, and should contain the main CTA link for the card. The style and color of the button should always be an 'arrow' style button stretched across the entire width of the card in light gray, which changes to the card color when it is hovered or focused on. If the card has more than one CTA to link to (a landing page and a brochure link, for example), the additional CTA link(s) should be added under the card description in the text content section of the card.

Contents and options:

  • CTA button: This element should use the standard classes for a small, light-gray arrow button. The expanded size and hover color are controlled automatically by the card CSS. There should only ever be one main CTA button per card in the footer.

Creating a Card Grid

Cards are most commonly displayed on the website in rows of 3 within a grid or carousel. This section will break down how to write the code for common types of cards and card containers.

Card Container

To start building a card grid, the parent element must be created first. The container element will wrap around all of the cards and determine how they are laid out on the page. Any parent container for cards will need the cards class to apply proper styles to the cards within. For ease of visualizing the different part of the code, the actual card HTML is left out of these examples and will be shown at later sections.

Cards

Once the parent container has been created, it can be filled with any number of cards, usually contained within the grid-item element, or within the column container in custom layouts. Each card then has 3-4 sections of content within it (described in the 'anatomy of a card' section of this guide). The card creation guide will go through some common options for each section of content and give code examples for each section.

Card Article

Each card is a card class <article> element, and should also have the hover-container class to allow for hover effects on images and a component-color class to determine color theme.

	            
HTML - Card article

	            
					<article class="card hover-container component-red">

						  [Card sections stacked inside...]

					</article>

Card Header

The card header is a card-header class <header> element inside of the card article, and should only be added if a header is appropriate (i.e. it is not a resource thumbnail card). This element should always include a card-corner class div with an icon or date and may optionally have the card-type class <div> as well.

Icon header with type label: The icon used in the card corner is determined in the icon class i element (refer to the icon documentation for help finding the icon name class).

	            
HTML - Icon header with type label

	            
	                <header class="card-header">

	                      <div class="card-corner">

	                            <i class="icon icon-star medium"></i>

	                      </div>

	                      <div class="card-type text-block outline">Type Label</div>

	               </header>

Date header with type label: Add the date class to the card-corner <div> and use the month and day <span>s for the text.

	            
HTML - Date header with type label

	            
	                <header class="card-header">

	                      <div class="card-corner date">

	                            <span class="month">Jan</span>

	                            <span class="day">01</span>

	                      </div>

	                      <div class="card-type text-block outline">Type Label</div>

	               </header>

Icon header over an image: Add the positioned class to the header to layer it over a subsequent simple image content section.

	            
HTML - Icon Header over image content

	            
					<header class="card-header positioned">

						  <div class="card-corner">

							    <i class="icon icon-star medium"></i>

						  </div>

					</header>

Card Content

Text and image content is added with 1-2 card-content class <section> elements in the body of the card. Image content is usually either a resource thumbnail, simple image, or logo; text content can be any combination of card type, card title, line, extra details, description, extra CTA links, and tags. The different image types are listed here along with all the possible text content additions.

Simple image: The image is added to a ratio class element (either an <a> link or a <div>) using a series of classes and an inline background-image CSS style. These images should always have a light overlay hover effect to help set them apart from text content.

			    
HTML - Image content - Simple image

			    
			        <section class="card-content no-padding">

			              <a href="linkURL" class="background-image overlay-light hover-no-overlay ratio sixteen-nine" style="background-image:url('imageURL');" role="img"></a>

			        </section>

Resource thumbnail: A standard resource thumbnail can be used for image content and should always link to the resource it is describing. This example uses the video class, which determines the icon generated. Other options for resource thumbnail element are: presentation, document, image, report, ebook, brochure, analyst-review, webcast, success-story, tech-guide, white-paper, dashboard, blog, or webpage.

			    
HTML - Image content - Resource thumbnail

			    
			        <section class="card-content no-padding">

			              <a href="linkURL" class="resource-thumbnail video background-image" style="background-image:url('imageURL');" role="img">

			                    <span class="cta">CTA text</span>

			                    <div class="ratio sixteen-nine"></div>

			              </a>

			        </section>

Logo: A logo can be added using the standard ratio class element and logo class <img> inside. The spacing-40 class is used to give a 40-pixel spacing buffer around the edges of the logo container. A thin line is automatically added under the logo to help separate it from the text content.

			    
HTML - Image content - Logo

			    
					<a href="linkURL" class="ratio sixteen-nine">

						  <img class="logo spacing-40" src="logoURL">

					</a>

Text content: This code sample includes all the possible text content additions in the order of how they should appear on the card. Note that the line and text-block elements do not need a color assignment, as this is determined by the component-color in the parent card article. For the card-details element, each line of details should be enclosed in a span element. Extra CTA text links should be enclosed in a <p> element to ensure proper spacing; if there are multiple extra CTA links, they should be separated with a <br> element within the same <p> element.

			    
HTML - Text content - All text sub-content options

			    
			        <section class="card-content">

			              <div class="card-type text-block outline">Type label</div>

			              <h3 class="card-title">Card Title Text</h3>

			              <div class="line tiny"></div>

			              <p class="card-details">

			                    <span>Extra details text line 1</span>

			                    <span>Extra details text line 2</span>

			              </p>

			              <p class="card-description">Card description text.</p>

			              <p>

							    <a href="linkURL">Extra CTA link text »</a><br>

							    <a href="linkURL">Extra CTA link text »</a>

						  </p>

			              <div class="tags text-block-group small">

			                    <div class="text-block" role="button">Card Tags 1</div>

			                    <div class="text-block" role="button">Card Tags 2</div>

			              </div>

			        </section>

The card footer is a card-footer class <footer> element inside of the card article and should always be included in the card when it has a CTA link. Card options

Once the card content has been determined, there are a few optional classes that can be set and/or added to change certain aspects of individual cards.

Color theme

Every card should have a color theme assigned with the component-color class, added to the class list for the card <article> element. Component color has options for red, blue, navy-blue, orange, green, black, dark-gray, gray, light-gray, and white. The chosen color affects the card header, card type, line, and button elements. Once a color is assigned, the component color can be overridden by adding a different color class to the individual element (for example, adding the class orange to the line element would make the line orange regardless of the component-color assigned to the card).

				
HTML - Card color themes

				
					<div class="flex-grid-sm-1 flex-grid-md-2 flex-grid-lg-3 cards start">

						  <div class="grid-item">

							    <article class="card hover-container component-red">
